## Element

[element.active]: Cor de fundo usada para o estado ativo de um elemento que deve ter uma cor de fundo diferente da superfície em que está inserido. Os estados ativos são acionados quando o botão do mouse é pressionado sobre o elemento, ou quando a tecla Enter ou outro ativador é pressionado.,
[element.background]: Cor de fundo usada para o fundo de um elemento que deve ter uma cor de fundo diferente da superfície em que está inserido. Os elementos podem incluir: botões, campos de entrada, caixas de seleção, botões de opção... Para um elemento que deve ter a mesma cor de fundo da superfície em que está inserido, use `ghost_element_background`.
[element.disabled]: Cor de fundo usada para indicar o estado desativado de um elemento que deve ter uma cor de fundo diferente da superfície em que está inserido. Os estados desativados são exibidos quando um usuário não pode interagir com um elemento, como um botão ou campo de entrada desabilitado.
[element.hover]: Cor de fundo usada para o estado de foco de um elemento que deve ter um fundo diferente da superfície em que está. Os estados de foco são acionados quando o mouse entra em um elemento ou quando um dedo toca em um elemento em uma tela sensível ao toque.,
[element.selected]: Cor de fundo usada para o estado selecionado de um elemento que deve ter um fundo diferente da superfície em que está inserido. Os estados selecionados são acionados quando o elemento é selecionado (ou 'ativado') pelo usuário. Isso pode incluir uma caixa de seleção marcada, um botão que pode ser ativado ou desativado, etc.

## Ghost Elements

[ghost_element.active]: Cor de fundo usada para o estado ativo de um elemento fantasma que deve ter a mesma cor de fundo da superfície em que está. Os estados ativos são acionados quando o botão do mouse é pressionado sobre um elemento, ou quando a tecla Enter ou outro ativador é pressionado.
[ghost_element.background]: Usado para o fundo de um elemento fantasma que deve ter o mesmo fundo da superfície em que está inserido. Os elementos podem incluir: botões, campos de entrada, caixas de seleção, botões de opção... Para um elemento que deve ter um fundo diferente da superfície em que está inserido, use `element_background`.
[ghost_element.disabled]: Cor de fundo usada para o estado desativado de um elemento fantasma que deve ter a mesma cor de fundo da superfície em que está inserido. Os estados desativados são exibidos quando um usuário não pode interagir com um elemento, como um botão ou campo de entrada desativado.
[ghost_element.hover]: Cor de fundo usada para o estado de foco de um elemento fantasma que deve ter a mesma cor de fundo da superfície em que está. Os estados de foco são acionados quando o mouse entra em um elemento ou quando um dedo toca em um elemento em uma tela sensível ao toque.
[ghost_element.selected]: Cor de fundo usada para o estado selecionado de um elemento fantasma que deve ter a mesma cor de fundo da superfície em que está. Os estados selecionados são acionados quando o elemento é selecionado (ou 'ativado') pelo usuário. Isso pode incluir uma caixa de seleção marcada, um botão que pode ser ativado ou desativado, etc.

## Icons

[icon]: Cor de preenchimento usada para a cor de preenchimento padrão de um ícone.
[icon.accent]: Cor de preenchimento usada para a cor de destaque de um ícone. Pode ser usada para indicar quando um botão de ícone alternável está selecionado.
[icon.disabled]: Cor de preenchimento usada para a cor de preenchimento de um ícone quando ele está desativado. Os estados desativados são exibidos quando um usuário não pode interagir com um elemento, como um botão de ícone.
[icon.muted]: Cor de preenchimento usada para a cor de preenchimento suave ou discreta de um ícone. Isso pode ser usado para exibir um ícone em um painel inativo ou para reduzir o peso visual de uma série de ícones.
[icon.placeholder]: Cor de preenchimento usada para a cor de preenchimento do espaço reservado de um ícone. Isso pode ser usado para exibir um ícone em um campo de entrada que desaparece quando o usuário digita o texto.

## Git

- `conflict`
- `conflict.background`
- `conflict.border`
  Indica algum tipo de conflito, como um arquivo alterado no disco enquanto estava aberto, ou conflitos de mesclagem em um repositório Git.

- `created`
- `created.background`
- `created.border`
  Indica algo novo, como um novo arquivo adicionado a um repositório Git.

- `deleted`
- `deleted.background`
- `deleted.border`
  Indica que algo deixou de existir, como um arquivo apagado.

- `hidden`
- `hidden.background`
- `hidden.border`
  Representa um estado oculto, como um arquivo que está escondido em uma árvore de arquivos.

- `ignored`
- `ignored.background`
- `ignored.border`
  Indica que algo está sendo ignorado deliberadamente, como um arquivo ou operação ignorada pelo Git.

- `modified`
- `modified.background`
- `modified.border`
  Indica um estado alterado ou modificado, como um arquivo que foi editado.

## Estado

- `error.border`

- `hint.background`

- `info`
